# Revido Mail — Design System

**Monochrome, pro-tool bones.** Ink on paper — black, white, and gray do the work; one dark-blue
accent appears sparingly (links, focus, selected, AI). Fast and keyboard-first underneath. Read this
before building any screen.

## Golden rule: tokens only

Style with **token utilities**, never arbitrary Tailwind values.

- ✅ `bg-primary`, `text-muted-foreground`, `border-border`, `rounded-2xl`, `bg-cat-newsletters/12`
- ❌ `bg-[#ff5a5f]`, `w-[327px]`, `text-[13px]`

The ESLint rule `tokens-only/no-arbitrary-values` flags violations. If you need a value that no token
covers, add a token to `src/styles/theme.css` rather than inlining it.

## Color tokens (semantic — auto light/dark)

| Token                         | Use                                       |
| ----------------------------- | ----------------------------------------- |
| `background`                  | app canvas (warm off-white / warm dark)   |
| `foreground`                  | primary text                              |
| `card` / `popover`            | raised surfaces                           |
| `muted` / `subtle`            | quiet fills, hover states                 |
| `muted-foreground`            | secondary text                            |
| `border` / `input`            | hairlines, field borders                  |
| `primary`                     | **ink** — the one loud action (black button)  |
| `accent`                      | **dark blue** — the single sparing accent     |
| `secondary`                   | neutral chips/buttons                         |
| `success` / `warning`         | **neutral** — status reads by label, not hue  |
| `destructive`                 | **red** — the one retained danger hue         |
| `info` / `ai`                 | **dark blue** — informational / AI marker     |

Every color has a matching `*-foreground`. Opacity modifiers work: `bg-primary/12`, `text-ai/70`.

## Category system (monochrome)

Nine categories, identified by **label, not hue**. `<CategoryChip>` / `<CategoryDot>` render neutral
gray by default; pass `active` to highlight the one focal category (e.g. the open category page) in the
blue accent. The `bg-cat-<token>` utilities still exist, but every stem now resolves to the same
neutral — color is no longer a per-category signal.

## Type

- **One family: Inter** (`font-sans`, the default). No display serif — everything, from screen titles
  to body, is Inter. Hierarchy comes from **size and weight**, not a second typeface.
- **Screen titles**: `text-lg font-semibold tracking-tight`.
- **The one big moment**: the Today greeting is `text-2xl font-semibold` — a screen gets a single title
  that's allowed to be large (see the loudness budget below).
- **Marketing hero**: `text-4xl sm:text-5xl lg:text-6xl font-medium tracking-tight` — big by size, but
  medium weight so it reads calm, not shouty.
- Sizes are the Tailwind scale plus `text-2xs` (0.6875rem) for tiny meta/labels.

> **Loudness budget.** A screen gets ONE big element; big = size + space, never weight + color.

## Shape & elevation

- Radius is tight and modern — crisp without going brutalist: `rounded-xl` (buttons/inputs), `rounded-2xl` (cards), `rounded-3xl` (hero/marketing).
- **Shadows are reserved for true overlays.** Resting surfaces — cards, buttons, toolbars — carry no
  shadow; a hairline border does the separating. Reach for `shadow-pop` only on things that actually
  float above the page (popovers, dialogs, command palette). Structure reads through borders and
  spacing, not a resting lift.

## The AI marker

AI output wears the **blue accent** — the same sparing color, so anything AI reads as "notable" without
a second hue. Use `<Sparkle/>` inline or `<AiTag/>` as a pill, and **reserve it**: mark AI the user can
act on or might be surprised by (drafts, summaries, suggested replies, agent actions) — not every
AI-touched pixel. A marker that's everywhere stops meaning anything.

## Motion

`tw-animate-css` is loaded — use `animate-in fade-in-0 zoom-in-95` etc. Keep motion quick and subtle
(150–250ms). Prefer optimistic, instant transitions; the product should feel _fast_.

## Token ramps & emphasis

Three tiers, generated by `tokens/build-tokens.mjs` (never hand-edit the output). Reach for the
**highest tier that says what you mean**: emphasis role first, semantic token next, raw ramp step
only when nothing else fits.

**1. Primitive ramps — real utilities.** Three families ship a 10-step OKLCH ladder (`50`→`900`,
perceptually even lightness) exposed as Tailwind utilities: `neutral` (ink→paper — `bg-neutral-200`,
`text-neutral-700`, does ~the whole UI), `accent` (dark blue, used sparingly), and `danger` (red,
destructive only). Ramps are theme-independent — they don't flip in `.dark`; only the semantic layer does.

**2. Semantics alias the ramps.** Roles like `text.primary`, `border.subtle`, `action.loud`, `accent.*`
point at ramp steps and split into `light`/`dark` sets. Components ask for a role, never a step.

**3. Emphasis — the hierarchy rule.** Loudness comes from **contrast, not color**:

| Tier       | Fills / roles                                        | For                                             |
| ---------- | ---------------------------------------------------- | ----------------------------------------------- |
| **loud**   | ink fill (`neutral.900`, `action.loud`)              | the **one** primary action per view             |
| **medium** | neutral surfaces & borders (`bg.raised`, `border.*`) | structure — cards, toolbars, most controls      |
| **quiet**  | muted text / faint fills (`text.muted`)              | everything that should recede                   |

> **One loud thing per view. If everything is emphasized, nothing is.** The blue accent is not a
> second "loud" — spend it sparingly on links, focus, selected, and AI.

**WCAG discipline.** Action fills take the AA-safe ramp step: the ink primary is AAA on paper. The
generator computes WCAG 2.1 contrast for every ramp step and text pairing on **every build** and writes
`contrast.report.json` — a role that drops below AA (4.5) prints a `fail` and is treated as a build error.

## Component usage

Twelve primitives. Match the component to the job, and lean on the emphasis rule for which _variant_
to reach for.

**Button** — variants `primary` · `secondary` · `outline` · `ghost` · `subtle` · `ai` ·
`destructive` · `link`; sizes `sm` `md` (`h-9 px-3.5`, the default) `lg` (`h-10 px-5 text-sm`)
`icon` (`size-9`) `icon-sm`. Controls run dense — inputs, selects, and the tabs list share the `h-9`
line so a toolbar reads as one tidy row.
_Use when_ triggering an action. `primary` is the **one loud action per view** (send, save) — at most
one on screen; `secondary`/`outline`/`subtle`/`ghost` step down through the medium and quiet tiers
for everything else; `ai` (sparkle) for AI actions; `destructive` for irreversible ones; `link` for
inline navigation.
_Don't_ stack two `primary` buttons in a view, use a Button to navigate (use a link/router `Link` with
`asChild`), or reach for `destructive` on anything undoable.

**Input** — single-line text/number/email field (pairs with `Label`, `Textarea` for multiline).
_Use when_ collecting one short freeform value.
_Don't_ use for a fixed set of options (that's Select/Radio) or a boolean (that's Checkbox/Toggle).

**Checkbox** — a boolean in a form, or **multi-select** in a list.
_Use when_ zero-or-more independent choices, or a single opt-in that's submitted with a form.
_Don't_ use for one-of-many (that's Radio) or for a setting that should apply the instant it's
toggled (that's Toggle).

**Radio** — pick **exactly one** from a small, visible set.
_Use when_ 2–5 mutually exclusive options that are worth showing all at once.
_Don't_ use for on/off (Toggle/Checkbox) or for long lists — past ~6 options, use Select.

**Select** — pick one from a longer or space-constrained list, collapsed into a trigger.
_Use when_ many options, or the choices don't earn permanent space.
_Don't_ use for 2–3 options you could show inline (Radio reads faster) or for multi-select.

**Toggle (Switch)** — an **instant on/off** setting.
_Use when_ flipping a boolean that takes effect immediately, with no save step (notifications on,
dark mode).
_Don't_ use inside a form that's submitted later (use Checkbox) or when the action needs confirmation.

**Badge** — variants `default` · `outline` · `primary` · `accent` · `success` · `warning` ·
`destructive` · `ai`.
_Use when_ surfacing **system-owned status** the user reads but doesn't edit: paid (`success`),
overdue (`destructive`), due soon (`warning`), AI-generated (`ai`). Non-interactive.
_Don't_ make it clickable or removable, and don't use it for user-managed labels — that's a Tag.

**Tag** — a **user-managed keyword**, typically removable.
_Use when_ the user adds/removes their own labels (folders, keywords) — carries an “×” to delete.
_Don't_ use for system status (that's Badge) or as a plain non-interactive marker. Badge = the system
tells you; Tag = you tell the system.

**Avatar** — a person/sender identity (use `ContactAvatar` to derive initials + color from a contact).
_Use when_ representing a real human or account, with an image fallback to `initials(name)`.
_Don't_ use as a generic icon chip or a category marker (use `CategoryDot`/icons).

**Divider (Separator)** — a thin rule between sections or list groups.
_Use when_ separating unrelated content where whitespace alone reads as too loose.
_Don't_ scatter dividers between every row (let spacing group things) or use one as decoration.

**Spinner** — an **indeterminate** in-progress indicator.
_Use when_ a short wait with unknown duration (inline button loading, a loading pane).
_Don't_ use when you know how far along you are (use `Progress`) or to mask layout that could load as
a `Skeleton` instead.

**Tooltip** — a short hint on hover/focus (use `SimpleTooltip` for the common label case).
_Use when_ naming an icon-only control or adding a brief keyboard-hint (`Kbd`) — supplementary only.
_Don't_ put essential info, interactive elements, or long copy in a tooltip (it's unreachable on
touch and by many AT users) — surface that inline or in a popover.
