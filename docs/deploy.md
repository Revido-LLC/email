# Deploy

How Revido Mail ships: staging topology, secret injection, and the CI gate.

## Topology

Three independently deployable services on Railway, plus managed Supabase:

| Service             | Repo path     | Railway config             | What it does                                                                                       |
| ------------------- | ------------- | -------------------------- | -------------------------------------------------------------------------------------------------- |
| `@revido/web`       | `apps/web`    | `apps/web/railway.json`    | The React SPA, served via `vite preview`.                                                          |
| `@revido/api`       | `apps/api`    | `apps/api/railway.json`    | The Hono API ‚Äî contract-doc, OAuth, AI endpoints.                                                  |
| `@revido/worker`    | `apps/worker` | `apps/worker/railway.json` | Background consumers: sync, enrichment, digests.                                                   |
| Supabase (Postgres) | ‚Äî             | ‚Äî                          | Managed. Schema lives in `packages/db`; migrate with `drizzle-kit` (`db:generate` / `db:migrate`). |

Each Railway service points its "Config File Path" at its app's `railway.json`. Keep the service's
Root Directory at the repo root ‚Äî pnpm workspace filtering (`pnpm --filter @revido/<pkg>`) resolves
correctly from anywhere inside the workspace, and the root directory is where the lockfile and
hoisted `node_modules` live.

## Secrets: Infisical

This repo never puts real secret values in `package.json` scripts, `railway.json`, or any committed
file. Instead, every service's Railway **start command** is wrapped with `infisical run --`:

```
infisical run -- pnpm --filter @revido/api start
infisical run -- pnpm --filter @revido/worker start
infisical run -- pnpm --filter @revido/web exec vite preview
```

`infisical run` fetches secrets for the configured project/environment (`INFISICAL_TOKEN`,
`INFISICAL_PROJECT_ID`, `INFISICAL_ENVIRONMENT` ‚Äî see `.env.example`) and injects them as process
env vars for the wrapped command, without ever writing them to disk. Locally, run the same wrapper
in front of any `pnpm dev`/`pnpm start` invocation, or fall back to a local `.env` (gitignored; copy
`.env.example`).

A root `infisical.json` scaffold documents the project settings shape without any real IDs. Fill in
`workspaceId` once the Infisical project exists; it holds no secrets itself. (The Infisical CLI reads
project settings from a dot-prefixed `.infisical.json` generated by `infisical init` ‚Äî rename/copy
this scaffold to that path once the project is provisioned.)

## `$PORT`

Railway assigns `$PORT` at runtime and expects the service to bind to it.

- `apps/api` and `apps/worker` already read `process.env.PORT` directly in `src/index.ts` ‚Äî no
  further wiring needed there.
- `apps/web` is served by `vite preview`, which now reads `Number(process.env.PORT) || 5173` from
  `apps/web/vite.config.ts` for both `server.port` (dev) and `preview.port` (prod serve).

**Caveat:** `apps/web/package.json`'s `preview` script hardcodes `vite preview --port 5173`, and a
CLI `--port` flag always wins over the config value. Editing service start scripts in `package.json`
is out of scope for this pass, so `apps/web/railway.json` calls `vite` directly ‚Äî
`pnpm --filter @revido/web exec vite preview` ‚Äî bypassing the `preview` script's hardcoded flag so
the config's env-driven port actually takes effect. A future pass should drop `--port 5173` from the
`preview` script so the plain `pnpm --filter @revido/web preview` works again.

## CI gate

`.github/workflows/ci.yml` runs on every push and PR: `pnpm install --frozen-lockfile`, then
`pnpm build && pnpm typecheck && pnpm lint && pnpm test`. All four must pass before merge.

## Cut-over note

Production (`email.revido.co`) ships from the **`revido-llc` remote's `main`**, not `origin/main`.
Staging/preview deploys can track `origin/main`, but any change intended for production needs to
land on `revido-llc/main` to actually ship.
